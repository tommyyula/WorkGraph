第一页：快速了解 workgraph.ai 做什么

Page 1: What workgraph.ai Does

面向客户、投资人、合作伙伴的快速版本。

A quick version for customers, investors, and partners.

01
workgraph.ai 不是员工监控软件，而是企业的 AI Work Intelligence Layer：看见真实工作 → 记住隐性知识 → 生成 SOP → 形成 Agent Skill → 优化流程与算力成本。

1. 解决的问题

看不见

管理者不知道真实工作过程、卡点和返工在哪里。

记不住

优秀员工的经验存在操作习惯、邮件、表格和系统路径里，离职就流失。

自动化不了

RPA 要工程师写脚本，通用 AI Agent 又不懂企业流程。

算力浪费

VM、GPU、AI 推理和存储按峰值配置，但缺乏成本归因。

2. 产品一句话

workgraph.ai 通过云桌面/VM 中的屏幕流、窗口事件、App/URL、邮件、共享文件和资源指标，自动生成项目进展、事件链、流程图、SOP 草稿、个人工作记忆和算力成本洞察。

3. 三层价值

L1 感知层：录屏 + OCR/VLM + WorkEvent ↓ L2 知识层：个人助手 + 企业知识图谱 + 项目记忆 ↓ L3 执行层：SOP → Skill → Agent 执行（后续版本逐步推进） ↓ L0 基础设施层：VM / GPU / AI 推理 / 存储成本洞察

4. 第一阶段销售表达

理解工作，而不只是记录活动。workgraph.ai 将云桌面中的真实工作过程转化为日报、项目进展、流程洞察、SOP 草稿和自动化机会。
workgraph.ai is not employee monitoring software. It is an AI Work Intelligence Layer: observe real work → capture hidden knowledge → generate SOPs → create agent skills → optimize workflow and compute cost.

1. Problems We Solve

Work is invisible

Managers do not see the real work path, bottlenecks, or rework.

Knowledge is lost

Expert knowledge lives in habits, emails, spreadsheets, and system paths.

Automation is hard

RPA requires engineers; generic AI agents lack enterprise workflow context.

Compute is wasted

VMs, GPUs, inference, and storage are provisioned for peaks without clear cost attribution.

2. One-Sentence Product Description

workgraph.ai analyzes screen streams, window events, App/URL data, emails, shared files, and resource metrics from cloud desktops or VMs to generate project progress, event chains, workflow maps, SOP drafts, personal work memory, and compute cost insights.

3. Three Layers of Value

L1 Sensing: recording + OCR/VLM + WorkEvent ↓ L2 Knowledge: personal assistant + enterprise knowledge graph + project memory ↓ L3 Action: SOP → Skill → Agent execution, delivered gradually in later versions ↓ L0 Infrastructure: VM / GPU / AI inference / storage cost insight

4. First-Phase Sales Message

Understand work, not just activity. workgraph.ai turns cloud desktop activity into daily reports, project progress, workflow insights, SOP drafts, and automation opportunities.

1. 新开发原则

核心原则：不做 18 个月大平台计划。先用 3–6 个月做出可演示、可试点、可销售 MVP。后续功能根据真实客户需求重新排期。
  • 只做第一批客户最能理解、最愿意付费的功能。
  • 所有功能必须围绕真实工作流程 demo，而不是平台愿景。
  • 用 AI 编程生成大量前后端基础代码、测试、文档和脚本。
  • 暂缓 marketplace、完整 agent execution、大规模连接器矩阵。
  • 每 2 周交付一次可演示版本。

1. New Development Principles

Core principle: Do not build an 18-month platform now. Build a demoable, pilot-ready, sellable MVP in 3–6 months. Plan future features based on real customer demand.
  • Build only what first customers can understand and pay for.
  • Every feature must support a real workflow demo, not just a platform vision.
  • Use AI coding to generate foundational frontend/backend code, tests, docs, and scripts.
  • Defer marketplace, full agent execution, and a broad connector matrix.
  • Deliver a demoable build every two weeks.
